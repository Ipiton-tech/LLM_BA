# Разработка технического задания

Инструмент возможно использовать в качестве помощника при разработке ТЗ, предварительно указав потребность / проблему / вводную составляющую и сформулировав конкретный запрос.

ИИ может предложить очевидное решение, которое можно взять за основу будущего ТЗ.

{% cut "**_Пример промта_**" %}{name=cutgroup}

> _Пишу задачу для разработчика 1С по созданию нового алгоритма. Запутался. Нужно чтобы алгоритм разбивал заказ на 2 РН. В первую помещал товар с приоритетом с этого склада, а во вторую весь остальной. Но что делать, если товар есть только на складе с приоритетом не ясно. Помоги описать._
>
>  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/94c73a69-b8b2-47f7-8805-7b9f297574d7)_

{% endcut %}

{% cut "**_Пример промта_**" %}{name=cutgroup}

> _Мне нужно создать требования к внутренней доработке 1С. Программисты знают систему, нужно описать верхнеуровнево. Сейчас у нас есть два склада: Основной и NV. В программе нужно показывать общий остаток по этим складам. При этом, когда создается РН нужно создавать РН на разные склады по приоритету._
> _Приоритет нужно задавать в формате: Матрица приоритетов для отгрузки со складов (ЮрЛицо / Клиент, Склад, ГруппаТоваров / Товар, Приоритет — 0, 1, 2). Если остаток есть на двух складах, то сначала выбираем весь остаток с одного склада, если там товар закончится, то во вторую РН добавляем остаток с 2 склада. Все урезки фиксируем во второй РН._
>
>  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/b8f85564-9da1-4e36-bce1-6ab9b545a72e)_

{% endcut %}
