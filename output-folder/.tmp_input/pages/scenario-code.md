# Анализ и документирование программного кода

<small>_**Статус:** <span style="color: #54BA7E; font-weight: 600;">✓ Используем в работе</span>_</small>

Основная проблема, которую призван решить инструмент — работа с кодом в системах 1С, где отсутствует документация, комментарии и шаблоны.

ИИ может выполнить анализ программного кода, инициировать автоматическую генерацию документации, сценариев тестирования и технических заданий.

## Практика применения

*   Использование ИИ позволяет понять логику работы обработки без участия разработчика, получить рабочие Excel-примеры, а также сформировать инструкции для пользователей, тестов и техническое задание на доработку.
*   Сгенерированные документы служат «рыбой» и значительно экономят время на подготовку пользовательской документации.

<br>

## Механика работы с инструментом

### **Шаг 1: Первичный анализ**

Загрузите в ИИ код обработки из 1С, скриншоты интерфейса, текст ошибки и сформулируйте конкретный запрос (промт).
<br>
> **Пример промта для первичного анализа**
>
>  _У меня есть обработка в 1С. Объясни языком для аналитика что делает обработка общими словами. Какие данные нужно заполнить и на что они влияют. Какие объекты создаются в 1С и какие проверки и параметры есть._
>
>  **Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/1c8d0558-6aad-4fe5-9b34-8631eb806135)
