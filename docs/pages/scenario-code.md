# Анализ и документирование программного кода

Основная проблема, которую призван решить инструмент — работа с кодом в системах 1С, где отсутствует документация, комментарии и шаблоны.

ИИ может выполнить анализ программного кода, инициировать автоматическую генерацию документации, сценариев тестирования и технических заданий.

## Механика работы с инструментом

### **Шаг 1: Первичный анализ**

Загрузите в ИИ код обработки из 1С, скриншоты интерфейса, текст ошибки и сформулируйте конкретный запрос (промт).
<br>

> _**Пример промта**_
>
>  _У меня есть обработка в 1С. Объясни языком для аналитика что делает обработка общими словами. Какие данные нужно заполнить и на что они влияют. Какие объекты создаются в 1С и какие проверки и параметры есть._
>
>  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/1c8d0558-6aad-4fe5-9b34-8631eb806135)_

### **Шаг 2: Генерация артефактов**

На основе уже проанализированного кода возможно сгенерировать практические артефакты:
<br>

{% list tabs %}

- Excel-файл

  Генерация Excel-файла для загрузки.
  <br>
  >  _**Пример промта**_
  >
  >  _Создай пример excel для загрузки, укажи какие столбцы являются обязательными и какой тип данных там должен быть._
  >
  >  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/1c8d0558-6aad-4fe5-9b34-8631eb806135)_

- Сценарий тестирования

  Генерация сценария тестирования.
  <br>
  >  _**Пример промта**_
  >
  >  _Напиши сценарий тестирования для пользователя по шагам, простым языком. Он должен включать в себя подготовительный этап, проверку основного позитивного сценария, а также кейсы для проверки обработки, дублей и ошибок._
  >
  >  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/1c8d0558-6aad-4fe5-9b34-8631eb806135)_
