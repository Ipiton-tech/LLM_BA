# 3.2 Оценка готового технического задания

<small>_**Статус:** <span style="color: #54BA7E; font-weight: 600;">✓ Используем в работе</span>_</small>

В рамках работы бизнес-аналитика ИИ используется как инструмент для **рецензирования уже написанного ТЗ или функциональных требований (ФТ)**, позволяя получить объективную обратную связь и выявить слабые места.

ИИ способен анализировать требования и акцентировать внимание на потенциальных рисках проекта.

## Механика работы

### 1. Напишите ТЗ и отправьте на анализ

Аналитик самостоятельно пишет ТЗ и использует ИИ для анализа и оценки материала.

{% note tip %}

**Промт:** Оцени описание технического задания.

**Ссылка:** [https://ai.ipiton.ru/s/f5c9eb54-64e8-4efc-b254-d345267a805c](https://ai.ipiton.ru/s/f5c9eb54-64e8-4efc-b254-d345267a805c)

{% endnote %}

### 2. Получите оценку

ИИ предоставляет оценку по шкале (например, от 1 до 5), указывая на сильные стороны, потенциальные риски и зоны для улучшения, что помогает повысить качество документации.

{% note warning %}

### ! Ориентиры по качеству ТЗ

*   **Оценка 3,5/5** — достаточна для отправки материала **внутреннему разработчику**.
*   **Оценка не ниже 4/5** — требуется для подготовки материала для **внешнего разработчика**, не знакомого с бизнес-процессами компании.

{% endnote %}

### 3. Доработайте ТЗ

Аналитик изучает рекомендации и дорабатывает ТЗ / ФТ.

---

## Итеративное улучшение

*   Вносятся те правки, которые считаются **объективно полезными**.
*   Обновленная версия ТЗ направляется на **повторную проверку ИИ**, чтобы оценить, как изменилась итоговая оценка и какие слабые стороны остались.
*   Цикл может повторяться требуемое количество раз, чтобы довести ТЗ до желаемого качества (например, с оценки **3,5/5 до 4,9/5**).

{% note warning %}

### ! Не все рекомендации нужно внедрять

ИИ склонен находить что улучшить, даже если в этом нет необходимости.

Чтобы исключить погрешность при оценке ТЗ, при последующей итерации с ИИ в промте необходимо **указать на игнорирование тех ошибок**, которые, по мнению бизнес-аналитика, незначительны.

{% endnote %}
