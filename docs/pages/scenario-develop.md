# 3.4 Разработка технического задания

**Статус:** <span style="color: #54BA7E; font-weight: 600;">✓ Используем в работе</span>

## Назначение

Инструмент возможно использовать в качестве помощника при разработке ТЗ, предварительно указав потребность / проблему / вводную составляющую и сформулировав конкретный запрос.

ИИ может предложить очевидное решение, которое можно взять за основу будущего ТЗ.

{% note tip %}

### Пример промта

Пишу задачу для разработчика 1С по созданию нового алгоритма. Запутался. Нужно чтобы алгоритм разбивал заказ на 2 РН. В первую помещал товар с приоритетом с этого склада, а во вторую весь остальной. Но что делать, если товар есть только на складе с приоритетом не ясно. Помоги описать.

{% endnote %}

## Риски использования

{% note warning %}

*   **Полная генерация ТЗ с нуля — неэффективна**, т.к. инструмент предлагает слишком абстрактные решения.
*   ИИ может сгенерировать «рыбу» ТЗ, но его качество значительно ниже, чем у человека и требует **серьезной переработки**. Без глубокого понимания бизнес-процесса результат будет неоптимальным.

{% endnote %}
