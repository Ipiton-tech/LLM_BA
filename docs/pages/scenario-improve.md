# Доработка технического задания. Анализ рисков

## Шаг 1: Генерация уточняющих вопросов

В рамках задачи по подготовке ТЗ на доработку функциональности инструмента, ИИ может выступать в роли ассистента при написании ТЗ, генерируя уточняющие вопросы, которые помогают выявить неполноту требований.
<br>

>  _**Пример промта**_
>
>  _Нужно доработать эту обработку. Необходимо добавить возможность по сохраненным настройкам производить автоматическую загрузку файлов с заранее заданной папки. После загрузки файлы должны удаляться. Составь ТЗ для разработчика. Во вложении формат, которому должно соответствовать ТЗ. Задай мне уточняющие вопросы, если это нужно._
>
>  _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/908d6084-cb22-4938-8a6c-c1bba821a86c)_

## Шаг 2: Выявление рисков

На основе предложенного ТЗ, ИИ помогает выявлять потенциальные риски проекта.
<br>

>  _**Пример промта**_
>  
>  _Какие риски нужно подсветить заказчику в рамках этого ТЗ._
>
> _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/fd1a0be2-c81c-461f-a0c6-2b733c89b358)_

## Шаг 3: Подготовка к обсуждению с заказчиком

ИИ возможно использовать для превентивной проработки слабых мест ТЗ и подготовки к обсуждению с заказчиком.
<br>

>  _**Пример промта**_
>
>  _На основании этих рисков составь письмо на заказчика с вопросами, которые нужно ему задать._
>
> _**Ссылка на промт:** [открыть](https://ai.ipiton.ru/s/fc81c226-1e22-425d-89d9-982ab8c72ee0)_
